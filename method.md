# Method: World-Model Planning for Test Point Placement Under Coupled Constraints

## 1. Problem Formulation

We formulate test point placement as a sequential decision process. Given a PCB board with known geometry, obstacles, and trace starting points, an agent places N test points one at a time. Each placement is a discrete selection from a set of valid candidate positions derived from the board's physical constraints.

The process terminates after all N test points are placed. The quality of the complete placement is evaluated by routing traces from starting points to test points and measuring total trace length, inter-trace spacing, and constraint satisfaction.

---

## 2. Why This Problem Is Hard

Test point placements are coupled through three constraint channels simultaneously:

**Spatial coupling.** Each placed test point creates an exclusion zone (minimum center-to-center spacing). Placing point A eliminates candidate positions for points B, C, D, and so on. The feasible set shrinks with every decision, and early placements can make later placements infeasible.

**Routing coupling.** The trace connecting a starting point to its test point occupies physical space on the board. This trace must maintain clearance from all other traces, obstacles, and board edges. Placing one test point determines a routing corridor that may block routing corridors for subsequent traces.

**Length coupling.** Each test point's position determines its trace length. All traces must ultimately have matched lengths (via post-processing). If one placement produces a very long trace, all other traces must be meandered to match, requiring additional board space. Extreme length disparity can make equalization infeasible.

These couplings mean that the quality of any single placement cannot be evaluated independently — it depends on all future placements. The interaction space grows combinatorially with trace count.

---

## 3. The Approach: Learned World Models

### 3.1 Core Idea

We learn a predictive model of the placement process — a **world model** — that captures how placement decisions cascade through the coupled constraint space. The agent uses this model to mentally simulate future placement sequences before committing to any action.

### 3.2 World Model

The world model is a latent dynamics model with three components:

**Representation model.** Maps the current board state (obstacle layout, placed test points, routed traces, remaining candidates) into a compact latent representation.

**Transition model.** Predicts how the latent state changes when a placement action is taken. This encodes the constraint dynamics: exclusion zone propagation, routing corridor occupation, and trace length consequences.

**Prediction heads.** From any latent state, predict the expected reward (constraint satisfaction, trace length, spacing quality) and whether the episode continues or terminates (all candidates exhausted).

The world model is trained from experience: the agent interacts with a placement simulator, and the model learns to predict observation sequences, rewards, and terminations from the collected trajectories.

### 3.3 Planning via Imagination

Once the world model is sufficiently accurate, the agent plans entirely in imagination:

1. From the current real state, encode into the latent space.
2. Simulate multiple candidate actions forward through the transition model for H steps (the **imagination horizon**), producing imagined trajectories of latent states and predicted rewards.
3. Evaluate each imagined trajectory using a learned value function that estimates long-term cumulative reward.
4. Select the action whose imagined trajectory yields the highest expected value.

This "imagine-then-act" mechanism is the key advantage over model-free methods. A model-free agent (e.g., PPO, DQN) must execute a placement in the real environment to observe its consequences. The world model allows the agent to foresee that "placing test point 5 here will block the corridor needed for test point 12" — without actually making that mistake.

### 3.4 Actor-Critic in Imagination

The agent trains two networks from imagined trajectories:

**Actor (policy).** Maps latent states to action distributions. Trained to maximize imagined cumulative reward. Uses the REINFORCE gradient estimator for discrete action spaces.

**Critic (value function).** Estimates the expected future reward from any latent state. Trained on imagined reward sequences using temporal-difference learning with slow-moving target networks.

Both actor and critic are trained purely on imagined data generated by the world model — no additional environment interaction is needed beyond what was used to train the world model itself. This makes the approach highly sample-efficient.

---

## 4. Constraint-Driven Action Space

The physical constraints naturally restrict the action space. The minimum test-point-to-test-point spacing (13mm) and test-point-to-board-edge spacing (14mm) define a finite set of valid candidate positions on any given board. At typical grid resolutions (5–7mm), this produces 50–200 candidates per board instance.

The agent selects from this discrete candidate set at each step. After each placement, candidates that fall within the new exclusion zone are removed. This dynamic masking ensures the agent only considers physically valid placements.

The candidate set size (~100–200) is comparable to action spaces in environments where world-model RL has been successfully applied (e.g., Atari, Crafter), making the problem tractable without action space decomposition.

---

## 5. Reward Structure

The reward combines per-step and episode-level signals:

**Per-step rewards** provide dense learning signal during placement. The agent receives positive reward for constraint-satisfying placements and negative reward for violations. An additional term rewards preserving future placement options (fraction of candidates remaining), discouraging greedy early placements that corner the agent later.

**Episode-end rewards** evaluate the complete placement via deterministic routing (A* pathfinding from starting points to placed test points). The reward reflects total trace length (minimize), minimum inter-trace spacing (maximize), routing feasibility (all traces must be routable), and trace length spread (keep similar for post-hoc equalization).

The per-step rewards address credit assignment — the agent knows immediately whether a placement is valid — while the episode-end rewards capture global quality metrics that depend on the full configuration.

---

## 6. Length Matching via Post-Hoc Equalization

Exact length matching is deferred to a deterministic post-processing step. After placement and routing, shorter traces are extended via serpentine meandering patterns to match the longest trace. This is a solved problem with established algorithms (dynamic programming for obstacle-aware meandering).

During placement, the agent receives only a soft penalty for trace length spread — the ratio of (max length − min length) to mean length. This steers the agent toward layouts where traces are roughly similar in length (within ~10–20%), ensuring that post-hoc meandering is feasible without requiring exact matching during the placement decision.

This decomposition simplifies the RL problem significantly. The agent optimizes placement quality; classical algorithms handle equalization.

---

## 7. Generalization Across Board Configurations

The agent must generalize to boards with different dimensions, trace counts, and obstacle layouts. Three mechanisms enable this:

**Spatial normalization.** All coordinates are normalized relative to board dimensions, producing a canonical representation regardless of physical size.

**Variable-length handling.** The candidate set and trace count vary per instance. Fixed-size representations with masking accommodate this variation.

**Training distribution.** The agent trains on a distribution of procedurally generated boards (varying dimensions, obstacle count/position, trace count) or a dataset of real board specifications. The world model learns constraint dynamics that transfer across configurations.

---

## 8. Hypothesis

**Model-based RL (world models) outperforms model-free RL (PPO, DQN) specifically on boards where constraints are tightly coupled** — many traces, small board, obstacles creating bottlenecks. On easy boards (few traces, large board), the advantage diminishes because lookahead is unnecessary.

This is a testable claim. The expected result: world-model agents show disproportionate improvement on constrained boards where model-free agents fail to anticipate cascading constraint violations.

---

## 9. Novelty

All prior work on RL for PCB design assumes fixed endpoints — the netlist is given, and the agent optimizes routing order or component placement within a known topology. This work addresses a different problem class: **generative co-design**, where the endpoints themselves (test point positions) are design decisions. The agent creates the topology rather than optimizing within one.

The world model's role is correspondingly different. In prior work, the world model predicts routing outcomes for a fixed design. Here, it predicts the feasibility and quality consequences of design decisions that shape the topology itself — a harder prediction task that requires capturing coupled spatial, routing, and length interactions.
